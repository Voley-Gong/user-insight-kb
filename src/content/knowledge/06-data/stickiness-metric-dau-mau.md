## 🎯 这个理论解决什么问题？

> "我们的DAU破百万了！"
>
> 三个月后，游戏关服了。

这不是段子，这是2023年中国手游市场上演了几十次的真实剧本。DAU（日活跃用户）是整个游戏行业最常被引用、也最常被误用的指标。投资人看DAU，老板看DAU，市场报告看DAU——但DAU本身告诉你的是"今天有多少人打开了你游戏"，而不是"这些人有多在乎你的游戏"。

一个DAU 100万但用户每天只打开1分钟的游戏，和一个DAU 20万但用户每天玩2小时的游戏，哪个更健康？答案不言而喻——但如果你只看DAU，你永远问不出这个问题。

**DAU/MAU 粘性指标**（Stickiness Ratio）解决的就是这个问题：它不看你有多少人来了，而看你的用户**有多频繁地回来**。

## 📐 核心理论

**一句话结论**：DAU/MAU 比值（又称"粘性指数"或"活跃系数"）衡量的是"在一个月内使用过产品的用户中，平均每天有多大比例会回来"，是判断产品真实健康度远比单看DAU更准确的指标。

这个指标最早由 Facebook 的数据团队 (Gebelia, 2009) 在内部报告中提出并推广。当时 Facebook 发现一个反直觉现象：一些第三方应用的 DAU 在增长，但用户时长和互动深度在下降——增长的是"路过"的用户，不是"留下"的用户。于是他们引入了 DAU/MAU 比值来区分**"流量型增长"**和**"粘性型增长"**。

在游戏行业，Seif El-Nasr et al. (2013) 在《Game Analytics》中将 DAU/MAU 定义为"玩家参与度的核心温度计"：

```
DAU / MAU = 当日活跃用户数 / 过去30天活跃用户总数

含义：在过去30天玩过你游戏的人中，今天有多少比例回来了？

┌──────────────────────────────────────┐
│  DAU = 1,000,000  MAU = 10,000,000   │
│  DAU/MAU = 10%                        │
│  → 30天内有1000万人来过               │
│  → 但每天只有10%回来                  │
│  → 这是"流量型"产品，不是"粘性型"产品  │
└──────────────────────────────────────┘
```

**行业基准**（游戏类产品）：

| DAU/MAU 区间 | 粘性等级 | 游戏类型代表 | 含义 |
|---|---|---|---|
| < 10% | 极低 | 一次性单机、轻量工具类 | 用户来了就走，几乎不回来 |
| 10% - 20% | 中低 | 买断制单机、部分超休闲 | 有一定回访但缺乏持续吸引力 |
| 20% - 30% | 中等 | 中度手游（SLG、卡牌） | 每月用户中约1/4每天回来 |
| 30% - 50% | 高 | 重度手游（MMORPG、MOBA） | 强日常习惯，核心玩家驱动 |
| > 50% | 极高 | 头部社交游戏、竞技大作 | 几乎是"日常刚需"级别 |

> ⚠️ 注意：DAU/MAU > 100% 在数学上是不可能的（因为DAU是MAU的子集），如果出现这种情况说明你的数据统计口径有问题。

## ⚙️ 底层机制

**类比**：DAU/MAU 就像一家健身房的**"到访率"**。

假设你办了张健身卡（= 成为了 MAU 的一员）。健身房老板统计：这3000个会员里，今天有多少人来了？

- 如果每天只有150人来 → 到访率 5% → 这家健身房大概率要倒闭
- 如果每天有900人来 → 到访率 30% → 非常健康
- 如果每天有1800人来 → 到访率 60% → 这是超级健身房

**关键洞察**：同样是"3000个会员"，到访率 5% 和 30% 的健身房是完全不同的生意。前者靠卖年卡赚钱（赌你不来），后者靠私教和附加服务赚钱（你来了才有钱赚）。游戏也一样——DAU/MAU 低的游戏靠拉新撑流水，DAU/MAU 高的游戏靠老玩家深度付费。

```
                   MAU (月活) = 10,000,000
                   ┌─────────────────────────┐
                   │  ○  ○  ○  ○  ○  ○  ○   │
                   │  ○  ○  ○  ○  ○  ○  ○   │  过去30天来过的所有人
                   │  ○  ○  ○  ○  ○  ○  ○   │
                   └─────────────────────────┘
                              │
                    今天回来了多少？
                              │
                   ┌─────────────────────────┐
                   │  ○  ○  ○               │
                   │  ○  ○  ○               │  DAU = 2,500,000
                   │  ○  ○  ○               │
                   └─────────────────────────┘
                              │
                    DAU/MAU = 25%
            "每月来的用户中，每天有1/4回来"
```

**为什么 DAU/MAU 比 DAU 更有洞察力？** 因为它同时包含了两个维度：

1. **广度**（MAU）：你的盘子有多大
2. **深度**（DAU/MAU 比值）：用户回来的频率有多高

DAU = 广度 × 深度。当你的 DAU 在涨时，你必须问：是因为 MAU 涨了（拉新成功），还是因为 DAU/MAU 涨了（粘性提升）？前者用钱就能买，后者才是产品的真实竞争力。

## 🎮 游戏设计应用

**真实踩坑故事**：

2022年，某国产SLG手游上线6个月，DAU从首月的200万降到80万，团队恐慌。老板拍桌子要求"拉新！买量！把DAU拉回150万！"。

市场团队花了2个月、砸了3000万买量，DAU确实回到了150万。但3个月后再次暴跌到60万——比之前还低。

为什么？因为他们只盯着DAU，没有看DAU/MAU：

| 时间 | DAU | MAU | DAU/MAU | 真实状况 |
|---|---|---|---|---|
| 上线首月 | 200万 | 350万 | 57% | 极高粘性，核心玩家驱动 |
| 第6个月（买量前） | 80万 | 400万 | 20% | 粘性下降，但盘子在涨 |
| 第8个月（买量高峰） | 150万 | 1200万 | 12.5% | **粘性崩塌**——买来的人根本不留 |
| 第11个月（买量停止后） | 60万 | 500万 | 12% | 买量用户全部流失，还带走了口碑 |

真正的问题从来不是"DAU跌了"，而是**"粘性跌了"**。DAU/MAU 从57%跌到12%，说明从"核心玩家每天必玩"变成了"大量路人一次性路过"。而买量不仅没解决问题，反而**稀释了粘性指标**——大量低质量用户涌入拉高了MAU，但这些人不会变成DAU。

**正确的诊断应该是**：

| 粘性指标变化 | 可能的设计原因 | 应该调查的方向 |
|---|---|---|
| DAU/MAU 持续下降 | 核心玩法缺乏长期目标 | 检查 endgame 内容是否充足 |
| DAU/MAU 突然下降 | 某次更新伤了体验 | 查看版本更新前后的留存对比 |
| DAU/MAU 稳定但 DAU 下降 | 没有新血进入，老玩家自然流失 | 拉新质量 + 新手引导优化 |
| DAU/MAU 稳定但 DAU 上升 | 产品粘性健康，且新用户能留存 | 继续保持，关注容量压力 |

**不同游戏类型的 DAU/MAU 基准对比**：

| 游戏类型 | 典型 DAU/MAU | 核心驱动力 | 粘性来源 |
|---|---|---|---|
| MOBA（王者荣耀、英雄联盟） | 40%-55% | 日常排位 + 社交 | 竞技快感 + 朋友一起玩 |
| MMORPG（原神、最终幻想14） | 35%-50% | 日常任务 + 角色养成 | 沉没成本 + 内容更新 |
| SLG/策略（三国志战略版） | 25%-40% | 联盟战争 + 领地扩张 | 社交责任 + FOMO |
| 卡牌/二次元（明日方舟） | 20%-35% | 活动周期 + 抽卡 | 收集欲望 + 角色情感 |
| 超休闲（合成大西瓜类） | 5%-15% | 无长期目标 | 单次体验即走 |

## ✅ 设计检查清单

- [ ] **你每天看的是 DAU/MAU 比值，而不仅仅是 DAU 绝对值？**
  （正：晨报第一行是"昨日 DAU/MAU = 32%，环比+2%" ✅ / 反：只看"DAU 80万，比上周少了5万" ❌）
- [ ] **DAU/MAU 按用户分层查看，而不是只看大盘？**
  （正：新用户DAU/MAU 8% vs 30天老用户DAU/MAU 55% ✅ / 反：只看混合值，不知道哪个人群拖了后腿 ❌）
- [ ] **你追踪 DAU/MAU 的趋势线，而不是只看某一天的快照？**
  （正：看30天趋势，识别"缓慢下降"或"断崖式下跌" ✅ / 反：今天比昨天低了就紧张，高了就开心 ❌）
- [ ] **买量期间你额外监控"自然量DAU/MAU"和"买量DAU/MAU"分开的指标？**
  （正：买量用户粘性单独追踪，发现买量用户DAU/MAU仅5% ✅ / 反：买量用户混入大盘，被高粘性老用户掩盖 ❌）
- [ ] **你定义了适合自己游戏类型的 DAU/MAU 健康基线？**
  （正：作为中度卡牌游戏，设定基线25%，低于20%触发预警 ✅ / 反：拿MOBA的50%标准要求自己的休闲游戏 ❌）

## 📊 数据验证方法

| 指标 | 怎么算 | 健康标准 |
|---|---|---|
| DAU/MAU | 当日DAU / 过去30天去重活跃用户数 | 看游戏类型，参考行业基准表 |
| WAU/MAU | 当周WAU / 过去30天去重活跃用户数 | 通常约为 DAU/MAU 的5-7倍 |
| 新用户DAU/MAU | 当日DAU中新用户 / 过去30天新用户 | 评估新手引导效果，通常<15% |
| 老用户DAU/MAU | 当日DAU中老用户 / 过去30天老用户 | 评估核心玩法吸引力，应>30% |
| 付费用户DAU/MAU | 当日付费用户DAU / 过去30天付费用户 | 付费用户粘性应显著高于大盘 |
| DAU/MAU 波动系数 | DAU/MAU 30天标准差 / 30天均值 | < 0.2 为稳定，> 0.4 说明粘性极不稳定 |

## 📖 新概念解析

| 概念 | 一句话定义 | 应用场景 |
|---|---|---|
| DAU | Daily Active Users，去重后的当日活跃用户数 | 衡量每日流量规模 |
| MAU | Monthly Active Users，去重后的过去30天活跃用户数 | 衡量用户盘子大小 |
| WAU | Weekly Active Users，去重后的过去7天活跃用户数 | 介于DAU和MAU之间的中期指标 |
| Stickiness Ratio | DAU/MAU 比值，核心粘性指标 | 判断产品健康度和用户参与深度 |
| 流量型增长 | DAU涨但DAU/MAU下降，靠不断拉新维持 | 不可持续，买量停了就崩 |
| 粘性型增长 | DAU涨且DAU/MAU稳定或上升，靠留存驱动 | 健康增长，有护城河 |
| 活跃定义 | "什么算活跃"的统计口径（登录vs完成行为） | 不同口径下DAU/MAU不可比 |

## 💡 个人洞察

我用一句话总结DAU/MAU的价值：**"DAU告诉你有多少人在门口，DAU/MAU告诉你有多少人真的住在里面。"**

在我见过的所有数据陷阱中，"唯DAU论"是杀伤力最大的一个。原因有三：

**第一，DAU 可以用钱买到，DAU/MAU 买不到。** 你花500万买量，DAU 涨了。但买来的用户如果第二天就走，你的 MAU 涨了、DAU 也短暂涨了，但 DAU/MAU 其实是下降的——因为分母（MAU）涨得比分子上涨后的回落更快。所以 DAU/MAU 是"去水分"指标，它对买量带来的虚高不敏感。

**第二，DAU/MAU 能提前预警设计问题。** 当一次版本更新引入了一个"赶客"的设计（比如难度暴涨、强制PVP、删减了某个受欢迎的日常玩法），DAU 可能不会立刻下降（因为MAU是30天滚动窗口），但 DAU/MAU 会在1-2周内持续走低。这就给你留出了响应时间——等DAU开始暴跌时，用户已经走光了。

**第三，最危险的模式是"DAU 涨但粘性降"的组合。** 这意味着你的产品正在从"核心用户驱动"变成"流量驱动"。短期内流水可能还过得去（因为MAU大了，总有少量人付费），但长期看，你的获客成本会越来越高，因为留存留不住人，必须不断买量补漏。最终的结果就是LTV < CAC——花100块买来的人，一辈子只产生80块价值。这个游戏就进入死亡螺旋了。

**实操建议**：在你的数据看板上，把 DAU 和 DAU/MAU 并排展示，用同一个时间轴。当你看到 DAU 和 DAU/MAU 的趋势线分叉（DAU涨但粘性跌），立即拉响警报。

## 🔭 下次玩游戏时观察

选一款你每天都会打开的游戏（比如原神、王者荣耀），做以下练习：

1. **估算它的 DAU/MAU**：你身边的玩家朋友中，注册过的人里有多少每天还在玩？你的直观感受是什么范围？
2. **识别粘性来源**：是什么让你每天回来？是日常任务？朋友组队？限时活动？还是纯粹的竞技快感？
3. **假想断奶实验**：如果这款游戏明天删掉所有"日常任务"和"限时活动"，只保留核心玩法（排位/剧情/探索），你还会每天打开吗？如果不会——它的 DAU/MAU 有多少是靠"设计绑架"而非"真实乐趣"支撑的？
4. **特别观察任务**：找一款你"下载了但一周没打开"的游戏，想想它做错了什么让你从"活跃用户"变成了"沉默MAU"——那个转折点是什么？一次卡关？一次糟糕的匹配？还是单纯的新鲜感过了？这就是 DAU/MAU 下降的微观故事。
