## 🎯 这个理论解决什么问题？

> "再给我五分钟，马上就好。"——这句话你已经说了三次了。
>
> 为什么无论是玩家还是开发者，都会系统性地高估自己的执行效率？

你以为这是拖延症？不，这是大脑底层的一个认知 bug：**规划谬误**。它让玩家觉得"一局只要20分钟"（实际45分钟），让开发者觉得"这个功能三天能做完"（实际两周）。

## 📐 核心理论

**一句话结论**：人类在规划任务时，会系统性地低估所需时间、成本和风险，即使有丰富的历史经验也无法自我修正。

Daniel Kahneman 和 Amos Tversky 在 1979 年首次提出"规划谬误"（Planning Fallacy）这一概念。他们在研究中发现：

- 学生完成论文的实际时间比预估平均**多花 55%**
- 即使学生被要求做"最悲观"的估计，仍然低估了实际时间
- 关键发现：**知道"我上次低估了"并不能阻止"这次再低估"**

核心机制是：规划时大脑只关注"最佳剧本"（best-case scenario），忽略了所有可能出错的外部因素。

## ⚙️ 底层机制

**类比**：规划谬误就像编译器只做 Happy Path 分析——

```
你的大脑规划逻辑：
  任务A（预计3天）
    → 任务B（预计2天）
    → 任务C（预计1天）
  总计：6天 ✅

现实执行逻辑：
  任务A（3天）
    → Bug阻塞（+2天）
    → 依赖库版本冲突（+1天）
    → 需求变更（+3天）
    → 任务B（2天）→ 联调失败（+1天）
    → 任务C（1天）→ 紧急热修插入（+2天）
  总计：15天 😱
```

**为什么会这样？** Kahneman 的解释是：

1. **系统1主导规划**：规划时大脑自动调取"最顺畅的记忆画面"，而不是统计真实分布
2. **锚定最佳案例**：你的估计被"一切顺利"这个锚点锁死了
3. **忽略基率信息**：你不参考"类似项目平均花了多久"，只关注"这次有什么不同"
4. **乐观偏差叠加**：你相信自己比别人更高效

## 🎮 游戏设计应用

**真实踩坑：某手游"快速一局"的承诺**

某竞技手游在宣传时强调"每局只需15分钟"，这是标准对局在完美条件下的时长。但玩家实际体验：
- 匹配等待：2-5 分钟
- 选人/BP 阶段：3-4 分钟
- 加载延迟：1-2 分钟
- 对局本身：18-35 分钟（因为有暂停、断线重连）
- 结算+社交：2-3 分钟

**实际单局时间：30-50 分钟**，是宣传时间的 2-3 倍。

后果：玩家的时间预期被打破，"我再打一局就睡"变成了凌晨两点的惨案。虽然有短期留存提升，但长期来看导致玩家产生"时间黑洞"的负面联想。

| 场景 | 规划谬误的表现 | 设计对策 |
|---|---|---|
| 对局时长 | 标注"15分钟"，实际30分钟+ | 显示"最近5局平均时长"的真实数据 |
| 抽卡预算 | 玩家预估"最多花200元"，实际花了2000元 | 设置硬性预算上限提醒 |
| 赛季目标 | 玩家高估自己能完成赛季任务的进度 | 显示"按你的历史速度，预计还需X天" |
| 功能开发 | 开发者预估"3天做完"，实际2周 | 参考类似任务的历史中位数，而非最佳案例 |

## ✅ 设计检查清单

- [ ] 时间相关提示是否用了**真实历史平均数据**？（而非理论最短时间）
- [ ] 是否有**预算/时间消耗的阶段性提醒**？（而非让玩家事后才发现）
- [ ] 赛季/活动进度是否显示了**基于个人速度的预估完成时间**？
- [ ] 预估开发工期时是否参考了**同类任务的历史中位数**？（而非拍脑袋）

## 📊 数据验证方法

| 指标 | 怎么看 |
|---|---|
| 单局实际时长 vs 宣传时长 | 对比均值差异，差异>30%说明规划谬误在误导玩家 |
| 赛季任务完成率 | 如果大量玩家最后3天才冲刺，说明前期预估过于乐观 |
| 付费转化 vs 预算预期 | 通过问卷对比"计划充值"和"实际充值"，差距越大说明预算规划谬误越严重 |

## 📖 新概念解析

| 概念 | 一句话定义 | 应用场景 |
|---|---|---|
| 规划谬误 | 系统性低估完成任务所需的时间和资源 | 时间提示、预算设计 |
| 参考类预测 | 不看个案，而是看同类项目的历史分布来做预估 | 工期规划、活动周期 |
| 乐观偏差 | 认为自己比平均水平更不容易遇到问题 | 玩家留存预估 |
| Happy Path 规划 | 只考虑一切顺利的最优路径，忽略异常情况 | 任何时间/资源估计 |

## 💡 个人洞察

规划谬误是游戏设计中**最被低估的认知偏差**。

大部分设计师关注的是"如何让玩家觉得好玩"，但很少有人关注"玩家以为自己要花多少时间 vs 实际花了多少时间"。这个差距直接影响玩家的**事后满意度**。

一个玩家觉得"被骗了时间"的玩家不会投诉，他只是默默流失。

更讽刺的是：**作为游戏开发者，我们自己也是规划谬误的重度受害者**。每个项目排期都过于乐观，每个"预估三天"最后都是两周。Kahneman 说过：**即使你知道规划谬误的存在，你也无法阻止自己犯同样的错误。**

唯一的解药不是"更谨慎地估计"，而是**用数据替代直觉**：参考历史中位数，而不是相信"这次不一样"。

## 🔭 下次玩游戏时观察

下次打开一款你常玩的游戏时，注意观察：

**你心里想的"我再玩一会儿就好"是多久？** 然后看一眼时间。再对比一下"每次关闭游戏时实际过了多久"。

如果你实际游玩时间是预期的 2 倍以上——恭喜，你亲身体验了一次规划谬误。
