## 🎯 这个理论解决什么问题？

> "给你100元" 和 "不扣你100元"，金额一样，但前者让你微笑，后者让你松口气。**为什么同样的结果，换个说法，玩家的情绪反应完全不同？**

游戏设计中最常犯的错误之一：精心设计了奖励，却用错误的措辞毁了体验。你写的是"登录奖励×7"，玩家读到的是"漏一天就亏了"。数值一模一样，感受天差地别——这就是**框架效应**。

## 📐 核心理论

**同一个客观信息，用不同方式呈现，会导致完全不同的决策和情绪反应。**

框架效应由 **Kahneman 和 Tversky（1979）** 在前景理论（Prospect Theory）中首次系统提出。他们发现，人在面对"获得"框架时倾向规避风险，面对"损失"框架时倾向追求风险——即使两个选择在数学上完全等价。

核心公式可以简化为：

```
客观价值 = 不变
心理价值 = f(措辞框架)
         → 正向框架（获得）：心理价值 × 1.0
         → 负向框架（损失）：心理价值 × 2.25（损失的心理权重约为收益的2.25倍）
```

这意味着：**你怎么说，比你说什么更重要。**

## ⚙️ 底层机制

框架效应就像**图片的相框**——同一幅画，放进金色雕花相框和放进黑色塑料框，观感完全不同。相框本身不改变画面内容，但它改变了你看画时的心理预期。

机制链路：

```
信息输入 → 情绪系统（System 1）自动评估"这是好事还是坏事"
                                    ↓
           损失框架触发威胁反应（杏仁核激活）→ 风险寻求
           获得框架触发满足反应（纹状体激活）→ 风险规避
                                    ↓
           理性系统（System 2）事后合理化 → "我选这个是因为..."
```

关键点：**System 1 在 System 2 介入之前就已经给信息贴好了"好/坏"标签**。这就是为什么即使你知道"不减分=加分"，那种"被扣分"的不爽感仍然挥之不去。

## 🎮 游戏设计应用

### 踩坑故事：一个"减分制"毁掉了整个赛季

> 某竞技手游的排位赛采用"胜加20分，负扣20分"机制。玩家每次输完一局看到"-20"的红色数字，挫败感极强。赛季中段数据显示：**连续输3局的玩家中，68%选择退出排位，而不是继续打**。
>
> 后来设计团队做了AB测试：将同样的ELO机制包装成"积分池"模式——初始分1500，赢了一局"积分+20"，输了一局显示"对手积分+20"（而不是"你-20"）。**数学上完全等价**，但退出率从68%降到了41%。

这就是框架效应的威力。以下是游戏设计中最常见的框架对照：

| 设计场景 | 负面框架（❌ 容易劝退） | 正面框架（✅ 更有动力） | 数学等价？ |
|---|---|---|---|
| 日常签到 | "连续中断，进度归零" | "今日签到，累计第X天" | ✓ |
| 战斗结果 | "你失败了，扣除10体力" | "挑战完成，获得经验！体力-10" | ✓ |
| 抽卡保底 | "还需要再抽70次才保底" | "已累计30次，保底进度60%" | ✓ |
| 赛季奖励 | "未达成目标，无法领取" | "再赢3局即可解锁" | ✓ |
| 时间限制 | "活动剩余不到2小时" | "还有2小时可以参加" | ✓ |
| 段位变化 | "你掉到了黄金" | "新赛季，目标王者！当前黄金" | ✓ |

### 设计原则

1. **默认使用"获得"框架**：把"不扣分"写成"积分+0"，把"没拿到"写成"还差X步"
2. **避免"归零"表述**：把"进度清零"改成"新一轮挑战开始"
3. **损失不可避免时，延迟展示**：先展示获得了什么，最后才提消耗
4. **进度条的方向决定情绪**：从0%→100%的"填满"比从100%→0%的"清空"积极得多

## ✅ 设计检查清单

| ✅ 正确做法 | ❌ 错误做法 |
|---|---|
| 进度条从0%向100%填充 | 进度条从100%向0%倒计时 |
| "再赢3局即可解锁" | "未达成，无法领取" |
| 先展示奖励再展示消耗 | 先展示消耗再展示奖励 |
| "今日累计签到第X天" | "中断后将重置进度" |
| 用"对手积分+20"替代"你-20" | 每次失败显示红色"-20" |

## 📊 数据验证方法

| 指标 | 测量方法 | 预期变化（正向框架 vs 负向框架） |
|---|---|---|
| 流失率（连续失败后） | AB测试两种结果展示方式 | 正向框架流失率降低15-25% |
| 任务完成率 | 对比"获得框架"与"损失框架"的任务文案 | 正向框架完成率高10-20% |
| 付费转化率 | 限时活动文案AB测试 | 正向框架（"限时优惠"）优于负向框架（"即将涨价"） |
| 皮肤满意度评分 | 相同皮肤，不同获取文案 | "恭喜获得"评分高于"消耗兑换" |

**推荐工具**：在游戏内嵌入AB测试框架，至少运行7天，收集1000+样本后再判断显著性。

## 📖 新概念解析

| 术语 | 一句话定义 |
|---|---|
| 框架效应（Framing Effect） | 同一信息的不同表述方式导致不同决策 |
| 前景理论（Prospect Theory） | Kahneman & Tversky 的决策模型：人面对收益保守、面对损失冒险 |
| 损失厌恶（Loss Aversion） | 损失的心理感受是等量收益的~2.25倍 |
| 获得框架（Gain Frame） | 强调"你能得到什么"的表述方式 |
| 损失框架（Loss Frame） | 强调"你会失去什么"的表述方式 |
| System 1 / System 2 | 双系统理论：System 1 快速直觉，System 2 缓慢理性 |

## 💡 个人洞察

框架效应是游戏UI设计中**投入产出比最高的优化点之一**。不改任何数值，只改措辞，就能显著改善体验。但很多团队忽视了它。

我在多个项目中观察到一种现象：**程序员出身的策划容易忽视框架效应**。因为数学上"-20分"和"对手+20分"完全等价，逻辑思维强的人会直觉地认为"这不重要"。但玩家的情绪系统不懂数学——它只懂"红字=坏，绿字=好"。

框架效应也有边界。当玩家**充分理解机制后**，框架效应会减弱——老玩家不会被措辞骗到，但他们仍然会被潜移默化地影响。所以框架效应的最佳应用场景是**新玩家引导期**，这时候认知负荷最高，情绪最敏感。

最后要注意：**框架效应不是欺骗**。你必须在所有框架下保持信息真实。把"扣10体力"改成"获得经验，体力-10"是合法的，因为你的确获得了经验；但把"掉率0.1%"写成"超高掉率"就是欺骗。框架效应的前提是：**同一事实的不同视角，而不是不同事实。**

## 🔭 下次玩游戏时观察

打开你最近在玩的游戏，关注以下三点：

1. **战斗失败画面**：第一个信息是"你失败了/扣XX分"还是"你获得了XX经验"？感受一下自己的情绪反应。
2. **日常任务文案**：是用"完成获得奖励"还是"未完成将重置"？
3. **限时活动倒计时**：是"剩余时间"还是"你将失去这次机会"？

尝试记录：哪些措辞让你感到焦虑或沮丧？哪些让你感到有动力？你正在亲身体验框架效应。
